# 윈도우 실행 파일 (더블클릭용)

터미널 명령을 몰라도 **파일을 더블클릭**하면 됩니다. 순서대로 하세요.

## 처음 한 번만

| 파일 | 하는 일 |
|------|---------|
| **설치.bat** | 파이썬 가상환경 만들고 aist 설치 + `config.yaml`/`persona.yaml`/`.env` 준비 |

설치 후 메모장으로 아래 3개를 열어 채웁니다:
- `config.yaml` — 플랫폼, 방송 시간 등
- `persona.yaml` — 캐릭터(이름·말투). 바꾸면 코어에 다시 넣기:
  `aist build-persona --conf Open-LLM-VTuber\conf.yaml --live2d <모델명>`
- `.env` — 키/토큰(LLM, 디스코드, OBS 비번, 플랫폼 등)

## 매번 방송할 때

| 파일 | 하는 일 |
|------|---------|
| **코어실행.bat** | 방송 코어(Open-LLM-VTuber) 실행 — 목소리·아바타 |
| **점검.bat** | 코어/OBS/플랫폼 채팅이 잘 붙는지 확인 |
| **테스트방송.bat** | 지금 한 방송만 (시작 수동, 종료 자동) — 비공개 테스트용 |
| **방송시작.bat** | 완전 자동 운영 (스케줄러가 켜고 끔) |
| **전체실행.bat** | 코어 + AI 방송인을 한 번에 (클릭 하나로) |
| **리포트.bat** | 직전 방송 리포트 + 컨텐츠 팩 보기 |

> **OBS** 는 미리 켜두거나, `config.yaml` 의 `obs.launch_if_not_running` 을
> 설정하면 자동으로 켜집니다.

## exe 파일이 꼭 필요하면

| 파일 | 하는 일 |
|------|---------|
| **EXE만들기.bat** | `dist\aist.exe` 생성 (PyInstaller) |

> `.bat` 만으로도 다 됩니다. exe 는 우리 자동화 레이어일 뿐이라, exe 를
> 만들어도 방송 코어와 OBS 는 따로 실행돼 있어야 실제 방송이 됩니다.

## 24시간 무인 운영

창을 계속 띄워두는 대신, 서비스로 등록하면 꺼져도 자동 재시작됩니다.
윈도우는 **작업 스케줄러**에 `방송시작.bat` 을 "시작 시 실행 / 실패 시 재시작"
으로 등록하거나, 리눅스 서버면 `deploy/install.sh` 를 쓰세요.
(자세한 건 `docs/INTEGRATION.md`)
